# MoguMap 決定記録

## 1. この文書の目的

この文書には、MoguMap開発で決定した内容と、その選択理由を記録する。

要件や技術構成を変更するときに、過去の決定理由を確認できるようにする。

まだ決まっていない内容は決定として扱わず、「保留事項」へ記録する。

## 2. 決定事項

### D-001：最初の対象地域を千葉県に限定する

- 状態：採用
- 記録日：2026-07-15

#### 決定内容

最初の完成範囲では、検索対象の地域を千葉県内に限定する。

地域検索には、ホットペッパーグルメAPIの中エリアを使用する。

#### 理由

最初から全国を対象にすると、地域選択や検索条件の実装範囲が大きくなる。

まず千葉県内に限定することで、飲食店を検索して一覧と詳細を表示する基本的な流れの完成を優先できる。

#### 将来の見直し

千葉県内を対象とした機能が完成した後、必要に応じて対象地域の拡大を検討する。

### D-002：npm workspacesによるモノレポを採用する

- 状態：採用
- 記録日：2026-07-15

#### 決定内容

MoguMapでは、npm workspacesを利用したモノレポ構成を採用する。

ルートの`package.json`では、次のディレクトリをworkspaceの対象にする。

- `apps/*`
- `packages/*`

現在のNext.jsアプリは`apps/web`に配置する。

#### 理由

アプリや共通パッケージを、1つのGitリポジトリとルートの設定で管理できるようにするため。

現時点では`apps/web`が主なアプリだが、今後ほかのアプリや共通パッケージが必要になった場合にも追加できる構成にする。

#### 注意点

モノレポ内には複数の`package.json`が存在する。

ルートの`package.json`はプロジェクト全体とworkspacesを管理し、`apps/web/package.json`はNext.jsアプリ固有の依存関係とスクリプトを管理する。

### D-003：apps/webでNext.js App RouterとTypeScriptを採用する

- 状態：採用
- 記録日：2026-07-15

#### 決定内容

フロントエンドのWebアプリを`apps/web`に配置する。

Webアプリでは、次の技術を使用する。

- Next.js
- React
- App Router
- TypeScript
- CSS Modules
- ESLint

App Routerのファイルは、`apps/web/src/app`に配置する。

#### 理由

ページとURLの関係をNext.jsのファイル構成で管理できるようにするため。

TypeScriptによって、データの型の間違いを開発中に見つけやすくするため。

#### 注意点

MoguMapのサーバー側APIにNext.jsのRoute Handlerを使うか、別のExpressアプリを使うかは、まだ決定していない。

### D-004：mainへ直接実装せずfeatureブランチで作業する

- 状態：採用
- 記録日：2026-07-15

#### 決定内容

`main`ブランチへ直接実装しない。

機能や変更内容ごとに`feature/`ブランチを作り、次の流れで`main`へ反映する。

1. featureブランチで変更する
2. 動作とGit差分を確認する
3. commitする
4. GitHubへpushする
5. Pull Requestを作成する
6. 内容を確認してmainへmergeする

commitメッセージとPull Requestは日本語で記述する。

#### 理由

動作する`main`を保ちやすくし、何を変更したのか確認してから反映できるようにするため。

無関係な変更や秘密情報が、意図せず同じcommitへ入ることを防ぎやすくするため。

### D-005：PostCSSをoverridesで暫定更新する

- 状態：暫定採用
- 記録日：2026-07-15

#### 決定内容

ルートの`package.json`に、次のnpm overridesを設定する。

`postcss`は`^8.5.10`を使用する。

現在実際にインストールされているバージョンは`8.5.18`である。

#### 理由

PostCSSの脆弱性`GHSA-qx2v-qp2m-jg93`は、バージョン`8.5.10`以上で修正されている。

一方、Next.js 16.2.10はPostCSS 8.4.31を依存関係として指定している。

`npm audit fix --force`ではNext.jsが意図しない古いバージョンへ変更される可能性があるため、実行せず、PostCSSだけをoverridesで暫定更新する。

#### 確認済みの結果

- `npm audit`：脆弱性0件
- Webアプリのbuild：成功
- Webアプリのlint：成功
- Webアプリの開発サーバー起動：成功
- トップページへのアクセス：HTTP 200
- `npm ci --dry-run --ignore-scripts`：成功

#### 注意点

Next.js 16.2.10が指定するPostCSSのバージョン範囲とは一致しない。

そのため、`npm ls postcss`では`invalid`と`ELSPROBLEMS`が表示される。

これは正式対応ではなく、動作確認を行ったうえで採用している暫定対応である。

#### 将来の見直し

Next.jsがPostCSS 8.5.10以上へ正式対応した後、overridesを削除できるか再確認する。
